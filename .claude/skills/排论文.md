# Word 论文排版

用户说"排论文""排版""跑流水线"时执行流程。说"微调""改XX格式"时编辑 build_generated.py。

## 完整流水线

### 1. 检查环境

```bash
python --version
python -c "import docx; from PIL import Image; print('OK')"
```

缺依赖则执行 `python -m pip install python-docx Pillow`。

### 2. 扫描文件

```bash
ls Templates/*.docx 2>/dev/null
ls Inputs/*.docx 2>/dev/null
```

- 如果没有 .docx 文件：告诉用户"请把模版 docx 放入 Templates/，内容 docx 放入 Inputs/，然后重新运行"
- 如果有文件：列出文件，让用户选号，或直接运行 `python run_pipeline.py` 让脚本自己交互
- 文件格式要求：`.docx`（Office 2007+ 格式），不接受 `.doc`

### 3. 运行

```bash
python run_pipeline.py
```

脚本会自动扫描文件、让用户选择、创建 `Outputs/{日期}_{内容名}/` 输出目录。

### 4. 验证结果

- 读 `Outputs/<最新目录>/格式提取.md`，核对段落数/字号/字体
- 读 `Outputs/<最新目录>/内容提取.md`，确认章节完整
- 确认最终 docx 已生成

### 5. 报告

简短汇报：X 段落 / X 章节 / X 参考文献，最终 N 页，输出目录位置。

## 微调排版

找到最新 `Outputs/*/build_generated.py`，按用户意图修改，然后：

```bash
python Outputs/<目录>/build_generated.py
```

### 修改速查

| 意图 | 位置 |
|------|------|
| 改正文字号/字体/行距/对齐/缩进 | `body()` 函数 |
| 改标题字号/居中 | `heading1/2/3()` 函数 |
| 参考文献字号 | `D['ref_size']` |
| 图片宽度 | `D['img_width']` |
| 正文前加分页 | 对应位置加 `doc.add_page_break()` |
| 表格字号 | `C()` 中的 `size` 参数 |

## 重要规则

- 所有格式值来自 `P{}`（模版提取）或 `D{}`（衍生计算），禁止硬编码
- 中文必须设 `w:eastAsia`，否则 WPS 回退明朝体
- A4 分页用双 cpl：拉丁文 cpl 和 CJK cpl 各自计算
- 改完 build_generated.py 别再跑 run_pipeline.py，会覆盖修改
- 每次运行生成独立目录，不会互相覆盖
