# Word 论文排版

## 触发条件

用户说"排论文""排版""跑流水线"时执行完整流程。说"微调""改XX格式"时编辑 build_generated.py。

## 完整流水线

### 1. 检查环境
```bash
python --version
python -c "import docx; from PIL import Image; print('OK')"
```
缺依赖则 `python -m pip install python-docx Pillow`。

### 2. 确认文件
检查 `Templates/` 和 `Inputs/` 下的 docx，与 `run_pipeline.py` 第19-20行的文件名一致。

### 3. 运行
```bash
python run_pipeline.py
```

### 4. 验证
- 读 `Outputs/格式提取.md`，核对段落数/字号
- 读 `Outputs/内容提取.md`，确认章节完整
- 确认 `Paper_Project/Program/build_generated.py` 生成成功

### 5. 报告
简短汇报：提取 X 段落 / X 章节 / X 参考文献，最终 N 页。

## 微调排版

读 `build_generated.py`，按用户意图改对应函数/参数，然后：
```bash
python Paper_Project/Program/build_generated.py
```

### 修改速查

| 意图 | 位置 |
|------|------|
| 改正文字号/字体/行距/对齐/缩进 | `body()` 函数 |
| 改标题字号/居中 | `heading1/2/3()` 函数 |
| 参考文献字号 | `D['ref_size']` |
| 图片宽度 | `D['img_width']` |
| 正文前加分页 | 对应位置加 `doc.add_page_break()` |
| 表格字号 | `C()` 中的 `size` 参数 |

## 重要规则

- 所有格式值来自 `P{}`（模版提取）或 `D{}`（衍生计算），禁止硬编码
- 中文必须设 `w:eastAsia`，否则 WPS 回退明朝体
- A4 分页用双 cpl：拉丁文 cpl 和 CJK cpl 各自计算
- 改完 `build_generated.py` 别再跑 `run_pipeline.py`，会覆盖你的修改
