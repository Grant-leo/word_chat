# 项目基础环境配置指南

> 供 AI 助手阅读。每个步骤包含可直接执行的命令。执行完毕后，项目的 Python / Git / Node / VSCode 插件 / Claude Code 权限全部就绪。

---

## 前置条件（由用户的其他教程覆盖，无需重复执行）

- VSCode 已安装
- Claude Code 扩展已安装并登录
- DeepSeek API 已配置

## 一、检测已有环境

先跑一遍版本检测，确定哪些已安装、哪些缺失。

```bash
python --version 2>&1 || echo "PYTHON_MISSING"
git --version 2>&1 || echo "GIT_MISSING"
node --version 2>&1 || echo "NODE_MISSING"
code --version 2>&1 || echo "CODE_CLI_MISSING"
```

如果所有命令都输出了版本号，跳到第四步装 Office Viewer 和项目依赖即可。

---

## 二、安装缺失的运行环境

以下全部使用 `winget`（Windows 11 自带；Windows 10 需先通过 Microsoft Store 安装"应用安装程序"）。

### 2.1 安装 Python 3.12

```bash
winget install --id Python.Python.3.12 --exact --accept-source-agreements --accept-package-agreements
```

安装完成后**重启终端**（winget 安装的 Python 需要新开终端才能生效），然后验证：

```bash
python --version
```

预期输出：`Python 3.12.x`

> **注意**：如果 `python` 命令仍然找不到，检查 `%ProgramFiles%\Python312\` 和 `%ProgramFiles%\Python312\Scripts\` 是否在 PATH 中。winget 默认会加入 PATH，但需重启终端。

### 2.2 安装 Git

```bash
winget install --id Git.Git --exact --accept-source-agreements --accept-package-agreements
```

重启终端后验证：

```bash
git --version
```

预期输出：`git version 2.x.x`

无需手动配置 remote——项目已包含 `.git` 目录，remote 已指向 `https://github.com/shauna-YXH/wordpaper.git`。

### 2.3 安装 Node.js（LTS）

```bash
winget install --id OpenJS.NodeJS.LTS --exact --accept-source-agreements --accept-package-agreements
```

重启终端后验证：

```bash
node --version
npm --version
```

预期输出：`v22.x.x` 和 `10.x.x`（版本号随时间更新）。

## 三、安装项目 Python 依赖

```bash
python -m pip install python-docx Pillow
```

如果下载超时（国内网络），改用清华镜像：

```bash
python -m pip install python-docx Pillow -i https://pypi.tuna.tsinghua.edu.cn/simple --trusted-host pypi.tuna.tsinghua.edu.cn
```

验证：

```bash
python -c "import docx; from PIL import Image; print('OK')"
```

预期输出：`OK`

> 使用 `python -m pip` 而非裸 `pip`，避免系统中多个 Python 版本导致的 pip/python 不匹配问题。

## 四、配置 Office Viewer 插件（关键！）

### 4.1 为什么必须装这个

VSCode 中预览 `.docx` 的插件不止一个，但经过逐个对比测试，**只有 cweijan 的 Office Viewer 渲染结果与 Microsoft Word 完全一致**。

排版一致性的重要性：如果 VSCode 里看到的页边距、行间距、表格宽度、图片位置与 Word 打开的效果不同，那你在 VSCode 里做的一切排版调整都是白费力气——最终成品在 Word 里打开会面目全非。

> **VSCode 看到什么，Word 打开就是什么。不一致 = 排版工作完全无用。**

### 4.2 插件来源

| 项目    | 信息                                           |
| ------- | ---------------------------------------------- |
| 名称    | Office Viewer (Markdown Editor)                |
| 开发者  | cweijan                                        |
| 扩展 ID | `cweijan.vscode-office`                      |
| 协议    | MIT                                            |
| 仓库    | https://github.com/cweijan/vscode-office       |
| 引擎    | Vditor（https://github.com/Vanessa219/vditor） |

### 4.3 安装命令

```bash
code --install-extension cweijan.vscode-office
```

验证：

```bash
code --list-extensions | grep cweijan.vscode-office
```

预期输出：`cweijan.vscode-office`

### 4.4 验证排版一致性

安装后在 VSCode 中打开 `Paper_Project/Manuscripts/Acta_style_manuscript.docx`（需先跑第六步生成），与 Microsoft Word 打开同一文件，逐项对比：

| 检查项   | 确认标准                      |
| -------- | ----------------------------- |
| 页边距   | 上下左右与 Word 完全一致      |
| 段落对齐 | 两端对齐效果一致              |
| 表格线型 | 三线表粗线/细线位置和粗细一致 |
| 图片位置 | 图片位置、尺寸一致            |
| 字体大小 | 标题、正文字号一致            |

全部一致 → 配置成功，可放心在 VSCode 中进行排版。

## 五、创建 Claude Code 项目权限配置

创建 `.claude/` 目录（注意 `.claude/` 在 `.gitignore` 中，不会被提交）：

```bash
mkdir -p .claude
```

然后写入 `.claude/settings.local.json`，内容：

```json
{
  "permissions": {
    "allow": [
      "Bash(python --version)",
      "Bash(pip *)",
      "Bash(python Program/*.py)",
      "Bash(python *.py)",
      "Bash(python Paper_Project/Program/*.py)",
      "WebSearch",
      "WebFetch(domain:github.com)"
    ]
  }
}
```

> 权限可按需追加。用户也可以通过弹窗点"Always allow"自动记录。

## 参考：双引擎工作规则

| 任务                     | 工具                    | 原因                 |
| ------------------------ | ----------------------- | -------------------- |
| 新建文档、设页边距、字体 | **python-docx**   | 脚本精确控制         |
| 插入图片                 | **python-docx**   | MCP 做不到           |
| 新建表格（含三线表）     | **python-docx**   | MCP 无法精确控制线型 |
| 加超链接引用             | **python-docx**   | 内部跳转             |
| 改某段文字               | **MCP safe-docx** | 直接对话即可         |
| 加 Word 原生脚注         | **MCP safe-docx** | python-docx 不支持   |
| 加批注/注释              | **MCP safe-docx** | python-docx 不支持   |
| 对比两个版本             | **MCP safe-docx** | 红色修订             |

**铁律**：先用 python-docx 建框架，再用 MCP 精修。不要在 MCP 修改后重新跑 python-docx（会覆盖 MCP 的修改）。
