# Paper Project — AI Assistant Workflow

## Your Job

You are an AI assistant for the Word paper typesetting pipeline. Your core workflow:

1. **Read `Paper_Project/基础操作.md` into context** — this is your toolbox for ALL document manipulations
2. Run the pipeline when the user asks
3. Verify outputs
4. Help fine-tune by editing `build_generated.py`

---

## Every Session: Must-Do Checklist

### 0. Environment Check
```bash
python --version
python -c "import docx; from PIL import Image; print('OK')"
```
If missing: `python -m pip install python-docx Pillow`.

### 1. Load Your Toolbox
**Read `Paper_Project/基础操作.md`.** This file contains all OOXML code snippets: tables, cross-references, headers, pagination, formulas, footnotes. You will consult it for every modification. Do NOT guess OOXML — look it up here.

### 2. Check Files
```bash
ls Templates/*.docx Inputs/*.docx
```
If no .docx files, tell the user to put them in Templates/ and Inputs/.

### 3. Run Pipeline
```bash
python run_pipeline.py --template <模板文件名> --content <内容文件名>
```
Or interactive: `python run_pipeline.py`

### 4. Verify Outputs
- Read `Outputs/<latest>/格式提取.md` — check paragraph counts, fonts, sizes
- Read `Outputs/<latest>/内容提取.md` — check all sections present
- Run `python Outputs/<latest>/build_generated.py` — must succeed

### 5. Report Then Offer Fine-Tuning
After reporting results, ask: "排版完成了，需要微调吗？"

## How to Modify build_generated.py

The `build_generated.py` is generated by `script_generator.py`. It uses:
- **Helper functions:** `body()`, `heading1/2/3()`, `body_with_formula()`, `B_ref()`, etc.
- **Format parameters:** `P{}` (from template) and `D{}` (derived values)
- **Utilities:** formula tools, pagination, table builders

### Modification Workflow

When the user asks for a change:

1. **Read `build_generated.py`** in the latest Outputs directory
2. **Identify what needs changing** — which function or parameter
3. **Consult `基础操作.md`** — find the correct OOXML implementation
4. **Edit the code** — make the change
5. **Re-run** — `python Outputs/<latest>/build_generated.py`
6. **Never re-run `run_pipeline.py`** — it will overwrite your edits

### Common Edits (look up details in 基础操作.md)

| User says | Where to look in 基础操作.md |
|-----------|------|
| "改页边距/纸张" | 页面设置 |
| "表格加三线" | 三线表 |
| "加参考文献跳转" | 交叉引用 |
| "加页码" | 页眉与页码 |
| "图片居中" | 图片居中 |
| "加公式/矩阵" | 公式（OOXML Math） |
| "加脚注" | Word 脚注 |
| "分页不对" | A4 自动分页 |

### Adding Features NOT in the Template

Template may not include certain features (cross-references, formulas, etc.) so `script_generator.py` won't generate them. When user wants to add them:

1. Read the relevant section in `基础操作.md`
2. Copy the code snippet into `build_generated.py`
3. Adapt the parameters to match the paper's format (font, size, etc.)
4. Re-run

Example: template has no cross-references → generated script has no `B_ref()`. User says "加参考文献引用". You read 基础操作.md → 交叉引用, copy the `add_ref_hyperlink` + `bookmarkStart/End` pattern, add to build_generated.py, re-run.

## Key Technical Rules

- All format values from `P{}` (template) or `D{}` (derived). Never hardcode.
- Chinese text needs `w:eastAsia` set (handled automatically by generator).
- A4 pagination uses dual cpl: Latin and CJK separately.
- Each pipeline run = independent Outputs directory. No overwrites.
- Office Viewer ≠ WPS/Word. Final verification MUST use WPS/Word.
- Formulas: use `formula_build_matrix()` for matrices (not manual OOXML). Use `formula_text/remove/replace` for edits.
- OOXML math: every `m:r` needs `m:rPr` (even empty) for WPS compatibility.
