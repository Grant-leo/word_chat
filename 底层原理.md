# 底层原理

## 一句话概括

**格式与内容分离。** 把学校模板 docx 里的字体/字号/行距/边距等格式参数全部提取出来，再把你的论文内容 docx 里的章节/段落/图片拆解出来，两者合并生成一个 python-docx 构建脚本，运行即得排好版的 docx。

## 为什么要这么设计

Word 排版的痛点不是"不会调格式"，而是**每次改内容后格式就乱**。LaTeX 解决了这个问题——内容和格式分离，但门槛高、模板少、中文支持差。

这个项目取中间路线：**用 Word 做模板，用代码做引擎。**

## 核心三个文件

```
format_extractor.py  ← 读模板，提取所有格式参数（字体/字号/行距/边距/页眉/封面...）
content_parser.py    ← 读内容，拆解出章节/段落/图片/参考文献
script_generator.py  ← 两者合并，生成 build_generated.py（纯 Python，可直接运行）
```

## 关键技术点

### 1. StyleResolver — 解决 python-docx 的 None 问题

python-docx 对样式继承处理很差：一个段落用了"正文"样式，字号继承自 base style，API 就返回 `None`。

StyleResolver 直接读 `styles.xml`，构建完整的样式继承树，逐级向上追溯补齐。最终所有 run 的 font/size/bold 全部非 None。

### 2. 双来源格式提取

模板里有两种信息：作者写在括号里的格式说明（"一级标题小三号，加粗，居中"），和实际 OOXML 里的渲染值。前者是**意图**，后者是**实际值**。两者融合，文字说明优先级更高。

### 3. 零硬编码代码生成

生成的 `build_generated.py` 里没有一个写死的值。所有格式参数来自两个字典：

| 来源 | 含义 |
|------|------|
| `P['body_font']` | 从模板提取的正文字体 |
| `P['body_size']` | 从模板提取的正文字号 |
| `D['ref_size']` | 衍生计算：`body_size - 2` |
| `D['img_width']` | 衍生计算：`min(text_width × 0.55, 4.2)` |

改模板就能改全套格式，不用改一行代码。

### 4. 中文字体（w:eastAsia）

`r.font.name = 'Times New Roman'` 只设置了西文字体。中文渲染需要额外设置 `w:eastAsia`，否则 WPS/Word 回退到明朝体。项目自动从模板中检测 CJK 字体并全局注入。

### 5. 双 cpl 分页

拉丁字符和 CJK 字符宽度不同（Times New Roman 约 0.42 倍字号，宋体约 1.0 倍）。分页估算时用两个 cpl（每行字符数），自动检测段落中是否包含中文，选择正确的 cpl。

### 6. 双验证

每个提取器独立运行两次，比对段落数/表格数/run 数。不一致时第三轮仲裁。防止 python-docx 解析的偶然不确定性问题。

## 与 LaTeX 的对比

| | LaTeX | wordpaper |
|------|------|------|
| 学习成本 | 高（学语法、配环境、调模板） | 低（放文件、说一句话） |
| 模板创建 | 写 .cls / .sty 文件 | 直接在 Word 里调格式 |
| 中文支持 | 需要额外的包和配置 | 原生支持，自动检测 CJK 字体 |
| 数学公式 | 极强 | 弱（python-docx 不支持公式） |
| 输出格式 | PDF | docx（Word/WPS 可直接编辑） |
| 参考文献 | BibTeX，自动排序 | 手动排列，悬挂缩进 |
| 版本管理 | 纯文本，git diff 友好 | 中间产物是 JSON/MD，可审计 |
| 微调方式 | 改源码重新编译 | 对 AI 说话，AI 改代码重跑 |
| 适用人群 | 理工科研究生 | 本科生、文科、不想学 LaTeX 的人 |

## 局限与改进方向

### 当前局限

- **公式段落高度估算不准**：`_ph()` 对数学符号段落可能低估行数
- **混合中英文段落**：当前是二选一（全 Latin 或全 CJK），极端混合情况可能偏 1-2 行
- **不支持数学公式渲染**：python-docx 本身不支持，需 OOXML 直写 MathML
- **参考文献需手动排列**：不像 LaTeX 的 BibTeX 自动排序

### 改进方向

- 逐字符加权计算混合段落高度（Latin 0.42 + CJK 1.0 按比例加权）
- 封面信息（姓名/学号/指导教师）支持从 JSON/YAML 配置文件填写
- 多模板对比：同一份内容对多个学校模板生成多份 docx
- MCP 集成：生成框架后用 MCP safe-docx 做精细编辑（脚注、批注）
- 更多高校模板内置支持
